This is an informatics research prototype that explores advances in cloud and social computing to collect and manage Family Medical Histories. The goal is to assess the possibility of moving the hosting and control of the process entirely to the patient and, in the process, enabling the connected distribution of the family history among those being described. In this experiment we'll be using versioned code hosting in GitHub and Google Code to deliver the family history application into the domain the patient's Google.com account. This experiment uses OAUTH version 2 to manage the authorization workflow, such that the family history might safely aggregate, and migrate between multiple cloud providers. This is expected to be further facilitated and commodotized as W3C identifies mechanisms and standards that enable OAUTH orchestration to be assisted by the web browser. See Hanes 2011, for a discussion of this topic. The participated nature of the prototype is extended to the code development in order to create a computational ecosystem the patient can engage. This second design criteria seeks to support a participated partnership with the biomedical facilities contributing to the patient's medical history. A module loading mechanism is included that builds on earlier experiments on code integration for image analysis and for the cancer genome atlas.
